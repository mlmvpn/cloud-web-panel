# پنل ابری

پنل مدیریت کانفیگ VLESS/Trojan روی Cloudflare Workers، برای کاربرانی که نمی‌توانند (یا نمی‌خواهند) از یک اپ اندرویدی خاص استفاده کنند — مخصوصاً کاربران آیفون. هرکسی با اکانت Cloudflare شخصی خودش وارد می‌شود، در چند کلیک پنل موردنظرش را روی همان اکانت دیپلوی می‌کند و کانفیگ‌هایش را می‌گیرد.

## استفاده (برای اکثر کاربران)

نیازی به نصب یا اجرای چیزی نیست. پنل همین الان روی این آدرس فعال است:

**https://mlm.freehosting.dev/**

مراحل کار:

1. وارد آدرس بالا شوید و یک حساب کاربری بسازید (فقط ایمیل و رمز عبور).
2. از داشبورد Cloudflare خودتان (`dash.cloudflare.com` → پروفایل → API Tokens) مقدار **Global API Key** را بردارید و همراه ایمیل حساب Cloudflare‌تان در پنل وارد کنید.
3. هرکدام از پنل‌ها (BPB، EDG، Nahan، MLM، Zeus) را که می‌خواهید با یک کلیک دیپلوی کنید — همه‌چیز روی اکانت Cloudflare خودتان بالا می‌آید، نه روی سرور مشترک.
4. کانفیگ‌ها را بگیرید و با دکمهٔ «کپی کل کانفیگ‌ها» در کلیپ‌بورد کپی کنید، بعد در اپ VPN خودتان (Shadowrocket، Streisand، V2Box، v2rayNG و مشابه) با گزینهٔ Import from Clipboard اضافه‌شان کنید.

همین. هر دستگاهی، هر جایی، بدون نصب اپ.

## چرا این ریپازیتوری منتشر شده

سورس کامل این پنل عمداً منتشر شده تا هرکسی بتواند ببیند دقیقاً چه اتفاقی روی سرور می‌افتد — مخصوصاً این نکته:

> حتی مدیر همین سایت هم نمی‌تواند Global API Key، ایمیل کلادفلر، یا کانفیگ‌های ذخیره‌شدهٔ هیچ کاربری را بخواند. نه از داشبورد، نه با دسترسی مستقیم به دیتابیس.

جزئیاتش در بخش «امنیت» پایین همین صفحه است. اگر می‌خواهید همین را روی هاست خودتان هم بالا بیاورید، بخش «اجرای مستقل» را ببینید.

## امنیت: رمزنگاری بر اساس پسورد کاربر

هر مقدار حساسی که این پنل ذخیره می‌کند — Global API Key، ایمیل Cloudflare، و حتی خودِ لینک‌های کانفیگ (`vless://...`, `trojan://...`) — با کلیدی رمزنگاری می‌شود که از **پسورد همان کاربر** ساخته شده (PBKDF2-SHA256، ۱۰۰٬۰۰۰ تکرار، نمک تصادفی مخصوص هر کاربر)، نه یک کلید ثابت سمت سرور.

این کلید فقط در حافظهٔ سشن، فقط در طول همان نشست لاگین‌شده وجود دارد و هیچ‌وقت جایی ذخیره نمی‌شود. نتیجه:

- یک دامپ کامل از دیتابیس (چه دزدیده‌شده، چه در اختیار خود مدیر سایت) بدون دانستن پسورد واقعی هر کاربر، هیچ چیز قابل‌استفاده‌ای لو نمی‌دهد.
- به همین دلیل «فراموشی پسورد» در این پنل وجود ندارد و نمی‌تواند وجود داشته باشد: با ریست پسورد، کلید قدیمی برای همیشه از بین می‌رود و داده‌های رمزشدهٔ قبلی دیگر قابل بازیابی نیستند. راه‌حل، اضافه‌کردن دوبارهٔ اکانت Cloudflare است، نه ریست پسورد.
- به همین دلیل هم این پنل **لینک اشتراک خودکار (subscription URL)** ندارد — چون رمزگشایی کانفیگ‌ها فقط داخل یک نشست لاگین‌شده ممکن است، و یک لینک عمومی که اپ‌های بیرونی بی‌وقفه آن را بخوانند، دقیقاً همان چیزی‌ست که این مدل امنیتی را بی‌معنی می‌کند. به‌جایش، کاربر از داخل پنل کانفیگ‌ها را کپی می‌کند.

این مدل در برابر دسترسی به دیتابیس محافظت می‌کند، نه در برابر یک سرور که در همان لحظه، درحالی‌که کاربر لاگین است، کاملاً هک شده باشد — همان محدودیتی که در هر اپ وب سمت‌سرور دیگری هم وجود دارد.

## اجرای مستقل روی هاست خودتان

- فقط PHP خام + MySQL؛ بدون Composer، بدون Node، بدون مرحلهٔ build.
- نیازمندی‌ها: PHP 7.4 به بالا با افزونه‌های `curl`، `openssl`، `pdo_mysql`، `mbstring`.

مراحل:

1. کل فایل‌های این ریپازیتوری را روی هاست آپلود کنید.
2. یک دیتابیس MySQL بسازید.
3. آدرس `install.php` را باز کنید و مشخصات دیتابیس را وارد کنید.
4. بعد از اتمام نصب، فایل `install.php` را حذف یا تغییرنام دهید.
5. اولین کسی که ثبت‌نام کند، به‌صورت خودکار مدیر سایت می‌شود (فقط برای مدیریت لیست IP تمیز کاربرد دارد).

## امکانات

- دیپلوی مستقل پنج پنل روی Worker های Cloudflare: **BPB**، **EDG (Edgetunnel)**، **Nahan**، **MLM** و **Zeus** (چندکاربره، با دیتابیس D1 اختصاصی).
- دریافت کانفیگ از هر پنل و کپی دسته‌جمعی یا تکی.
- تنظیمات اختصاصی هر پنل (Proxy IP، پورت‌ها، پروتکل) و مدیریت کاربر برای Nahan/MLM/Zeus.
- ابزار مدیریتی «ترکیب با IP تمیز»: یک گروه کانفیگ را می‌گیرد و برای هر IP موجود در لیست ادمین، یک نسخهٔ جدید با همان SNI/Host اصلی می‌سازد.
- نام‌گذاری تصادفی و غیرقابل‌حدس برای ساب‌دامین/Worker و کموفلاژ SNI، برای عبور بهتر از فیلترینگ.
